## Реализация модели конструктивных действительных чисел

Конструктивное действительное число представляется не как просто сохранённое значение, а как объект, который умеет выдавать приближение с заданной точностью epsilon.

Это выражено через интерфейс:

```
type Real interface {
    Approx(eps float64) (float64, error)
}
```

Таким образом, любое число в данной модели должно поддерживать получение приближённого значения по запросу.

### Что реализовано

- интерфейс Real;
- создание константного конструктивного числа через NewConstant;
- создание конструктивного числа из функции приближения через NewFromApprox;
- арифметические операции:
  - Add;
  - Sub;
  - Mul;
- сравнение чисел через Compare;
- обработка некорректного значения epsilon;
- тесты для базовой функциональности, арифметических операций и сравнения.

## Реализация генетического алгоритма

Генетический алгоритм представляет собой цикл:
- Начальная инициализация популяции (выполняется только один раз)
- Вычисление приспособленности
- Отбор 
- Скрещивание 
- Мутации

Генетический алгоритм реализован как гибкий и расширяемый движок, работающий с любыми числовыми типами

### Основные компоненты

**Engine:**\
Управляет pipeline'ом, выполняет сам алгоритм

**Selection:** 
- Tournament: турнирный отбор, выбирающий лучшую особь из случайной группы.
- Ranking: ранговый отбор, который позволяет самым приспособленным особям не так сильно доминировать над остальными.

**Crossover:**
- SBX (Simulated Binary Crossover): имитирует поведение одноточечного кроссовера для вещественных чисел, позволяя контролировать среднее и расстояние между потомками и родителями.
- BLX (Blend Crossover): создает потомков в расширенном диапазоне вокруг родителей.

**Mutation:**
- GaussianMutator: Добавляет к гену случайную величину, распределенную по нормальному закону, с дисперсией зависимой от границ.

**Replacement (Замена популяции):**
- Elitism: Гарантирует выживание N лучших особей из предыдущего поколения.
- Absent: Полная замена старой популяции новыми потомками.


### Особенности реализации 

- возможность находить как max, так и min функции (через параметр Weight)
- возможность расширить логику через реализации новых поведений в пакетах selection, mutation, replacement, crossover
